# PR Ready

Generated by `agent-workflow-kit`. Edit the kit template,
`agent-workflow.config.json`, or a repo overlay, not this rendered file.

Read `AGENTS.md` first, then follow `.agents/skills/pr-ready/SKILL.md`.

Use `/pr-ready` in one of two modes:

- Current branch preflight before opening or requesting review for your own PR.
- Explicit PR mode with `/pr-ready --pr <number>` before requesting review,
  marking ready, approving, or merging an existing PR.

Configured defaults:

- Base branch: `master`
- Issue provider: `linear`
- Issue key pattern: `[A-Z][A-Z0-9]{1,9}-[0-9]{1,7}`
- Verification before PR-ready: `check, test, build`
- Current-head PR attestation required: `true`

Do not mark a PR ready, request reviewers, approve, or merge unless explicit PR
mode returns `ACCEPT-READY` and any required current-head readiness attestation
has been recorded in the PR body.
