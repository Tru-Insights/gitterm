Commit and Log to Linear

Commit changes and associate with a Linear issue.

Workflow

1. Review Changes

  • Run git status to see what's changed
  • Run git diff --staged and git diff to review the changes

2. Ask Which Linear Issue (Before Committing)

  • Fetch in-progress issues: linear issue list --team FUL --state started --sort priority --limit 10
  • Use AskUserQuestion to present options:
    • First option: "Create new issue" — for new work not yet tracked
    • List each in-progress issue (format: "FUL-XXXX: Title")
    • Last option: "Skip - just commit" — for trivial changes
  • If "Create new issue" selected, ask for the issue title

3. Create New Issue (if selected)

  • Create the issue BEFORE committing:
linear issue create --team FUL --title "TITLE" \
  --start --assignee self --no-interactive
  • Parse the issue ID from the output (e.g., "FUL-1489")
  • The --start flag sets state to "In Progress"

4. Commit

  • Stage the appropriate files (prefer specific files over git add -A)
  • Create a commit with the Linear issue ID prefixed:
    • Format: FUL-XXXX: Clear, concise message
  • Get the new commit SHA from git rev-parse HEAD

5. Log Commit to Linear Issue

  • Format commit link: [short-sha](https://github.com/ORG/REPO/commit/full-sha)
  • Fetch current issue description and append under a "## Commits" section
  • Update: linear issue update FUL-XXXX --description "..."
