# Checkpoint

Generated by `agent-workflow-kit`. Edit the kit template or
`agent-workflow.config.json`, not this rendered file.

Use `.agents/checkpoint.md` as the shared tactical checkpoint for this repo.
This is a resume handoff only. It is not a human session log, agent memory, or
personal preference store.

## Resume Check

At the start of a session, after compaction, or when the user asks to resume:

1. Read `.agents/checkpoint.md` if it exists.
2. Treat it as lower priority than the user's current request and `AGENTS.md`.
3. Do not overwrite it unless the user asks to checkpoint or save progress.

## Write Workflow

When the user asks to checkpoint:

1. Gather branch, recent work, hot files, verification state, and the immediate
   next step.
2. Create parent directories if needed.
3. Overwrite `.agents/checkpoint.md`.
4. Keep it short enough for a new session to read quickly.

Use this structure:

```markdown
# Resume: gitterm-v2

**Checkpointed:** YYYY-MM-DD HH:MM **Branch:** branch-name

## Just Did

One sentence describing the last meaningful action.

## Immediate Next Step

Specific action with file paths and commands where useful.

## Recent Files

- `path/to/file` - why it matters

## State

- Uncommitted changes, PR state, blocker, or verification state.

## Context

Optional short context if a fresh session would otherwise be confused.
```
